專題介紹與開發紀錄

「北屯區 vs 竹東鎮居住比較」專案，除了單純的數據呈現，我也嘗試加入了一些視覺設計與互動效果，以下是各部分的開發過程分享。

設計靈感與排版

動態背景設計

為了讓網頁看起來不那麼死板，我在背景加入了一些緩慢漂浮的漸層色塊，再蓋上一層毛玻璃 (Backdrop-filter) 濾鏡效果。這種處理方式能讓畫面多一點層次感，同時保持文字的易讀性，不會搶走主要內容的焦點。

卡片式 (Bento便當盒設計法) 排版

在處理大量數據對比時，我選擇了近期蠻流行的 Bento Grid(便當盒設計法)。將各項數據切分成大小不一的圓角卡片，除了在視覺上比較乾淨俐落，在適應手機版本 (RWD) 時也很方便做網格調整。

日夜模式 (深色主題) 切換

考量到閱讀舒適度，我在導覽列加入了日夜模式切換功能。點擊之後，除了背景和文字顏色更換，我也特別調整了圖表的配色，以及將地圖底圖一併切換為夜間模式，讓整體的深色主題更為一致。

圖表與地圖互動

雷達圖 (Radar Chart) 資料視覺化

針對居住評估的各項指標（如生活機能、房價、交通等），我用 Chart.js 實作了雷達圖。比起傳統的數據表格，將兩地的數據疊加後，讀者能更直覺地比較出北屯區與竹東鎮各自的面向。

實際地圖整合與圖資切換

在交通探索的區塊，我利用 Leaflet.js 串接了地圖服務。除了標示出捷運綠線與台鐵內灣線的位置，也結合了點擊地標後平滑移動並顯示詳細圖文的功能，讓使用者在觀看時能有更具體的地理空間感。

篩選不同生活型態 (Persona)

由於每個人的居住需求不同，我實作了一個簡單的「生活型態選擇器」：設計了頂客族、小家庭和退休族的情境。使用者點擊對應卡片後，系統會切換對應的文案，希望讓不同族群都有些參與感。

簡易投票與橫條動畫

在網站下方的總結處有一個簡易投票系統。點擊選項後，會以動畫方式展開進度條呈現結果比例，做為體驗的最後收尾。

核心區塊與技術筆記

針對 index.html 中的幾個主要區塊，簡略紀錄開發過程與前端實作方式。

01

首頁：滿版雙區域設計

首頁版面：一進網站就用滿版視窗將畫面分為左右兩半，配上簡單的進場動畫，幫助讀者快速掌握接下來要對比的兩個行政區。
實作方式：採用 Flexbox 達成響應式對半佈局。背景則是繪製了動態漸層光球，最後加上 CSS 毛玻璃 (backdrop-filter) 創造層次。
02

資料呈現：Bento 式排版

排版規劃：把生硬的人口、房價數據抓出來，單獨做成一張一張的卡片。為了不讓畫面太枯燥，加上了簡單的雙色進度條作為點綴。
實作方式：主要仰賴 CSS Grid 的網頁佈局，利用 grid-column: span 去編排卡片大小，配上一點透明度的邊框。
03

購屋情境：2000萬預算能買哪裡

情境帶入：把單坪的房價轉化為有實感的購物情境：「預算兩千萬在北屯買小兩房、在竹東買四房」，這樣比單純列出房價會更有感一點。
實作方式：使用雙欄排列，並以偽元素 (pseudo-elements) 安排分隔線與標籤。項目則是用原生的 <ul> 搭配客製化標籤樣式。
04

受眾劃分：使用者 Persona 切換

角色設計：設計了竹科上班族、頂客族和退休長輩三個常見的目標客群。可以切換並查看對應的生活建議。
實作方式：利用 JavaScript 監聽按鈕點擊，並抽換下方容器內的 HTML 內容，加上 CSS 過渡動畫讓切換過程不會太生硬。
05

在地探索：地圖 API 串接

地圖整合：放入可縮放拖曳的動態地圖，代替死板的地圖截圖；點擊旁邊的目的地卡片，地圖就會自動移動並開啟該標籤的照片與說明。
實作方式：串接 Leaflet.js。自己將座標寫死在陣列裡，用 JS 在地圖上畫線 (Polyline)，並控制中心點的飛越 (flyTo) 特效。
06

資料比較：動態長條圖

動態回饋：通勤時間如果只放個數字會有點無聊，所以在切換園區或市區時，底下的長條圖會隨著數值改變長短。
實作方式：透過監控 HTML <select> 變更，用 JS 更新進度條 CSS 裡的 width。
07

網頁收尾：雙層進度條與真實數據庫 (Google Sheets)

體驗總結：捨棄傳統的雷達圖表，改用乾淨直覺的雙層進度條歸納兩地特色。不僅如此，投票功能不再只是前端動畫，而是真正串接了資料庫，讓讀者點選後能即時看見所有社群參與者的真實意向。
實作方式：資料視覺化完全使用原生的 HTML 與 CSS Flexbox 排版，去除了外部圖表套件的依賴。後端則是利用 Google Apps Script (GAS) 建立免費的 API，將 Google 試算表轉化為輕量級的 JSON 資料庫。當讀者按下投票按鈕，前端會透過非同步 fetch 請求將資料送回試算表，並即時運算最新的得票佔比回傳至網頁更新進度條。
