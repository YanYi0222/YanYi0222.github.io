設計系統與基本元素

Design System & Core Elements

色彩使用

北屯橘紅 (Beitun)

#C67053

象徵都市的繁華、活力與房市的熱度。帶有一點溫暖的磚紅色調，呼應新興重劃區的建築意象。

竹東深綠 (Zhudong)

#5C776B

代表自然環境、客家山城與悠閒的生活步調。沉穩的墨綠色能讓人聯想到山林與寧靜的居住氛圍。

曜石黑 (Text Main)

#1D1D1F

作為深色模式的主色調與文字基底，比起純黑更能降低視覺疲勞，提供如雜誌般頂級的閱讀體驗。

字體使用

Ag

Cormorant Garamond (藝術點綴)

用於大字體的數字排版與序號（如 01, 02），賦予頁面優雅與高級感，打破純中文排版的單調。

標題

Noto Serif TC (思源宋體)

應用於所有的段落大標題 (H1, H2)，宋體的筆觸能帶來穩重、具文化底蘊的視覺感受，與雜誌風排版完美契合。

內文

Noto Sans TC (思源黑體)

網頁主要的內文閱讀字體。無襯線的設計在螢幕顯示上具備絕佳的清晰度與易讀性，確保長篇文章也能舒適閱讀。

互動設計與排版策略

Interactive Design & Layout Strategy

動態背景與毛玻璃

為了讓網頁看起來不那麼死板，在背景加入緩慢漂浮的漸層光球，再蓋上一層毛玻璃 (Backdrop-filter) 濾鏡效果。這種處理能讓畫面多一點層次感，同時保持文字的易讀性。

Bento 便當盒設計法

在處理大量數據對比時，選擇 Bento Grid 將各項數據切分成大小不一的圓角卡片，視覺上乾淨俐落，且在適應手機版本 (RWD) 時也很方便做網格堆疊調整。

日夜模式 (Theme) 切換

考量閱讀舒適度，實作了日夜模式功能。點擊後除了背景更換，也特別調整了圖表配色，以及將地圖底圖一併切換為夜間模式，讓整體的深色主題更為一致。

核心區塊開發歷程

Architecture & Development Timeline

01

首頁：滿版雙區域設計

視覺規劃：一進網站就用滿版視窗將畫面分為左右兩半，配上進場動畫，幫助讀者快速掌握接下來要對比的兩個行政區。
實作方式：採用 Flexbox 達成響應式對半佈局。背景繪製動態漸層光球，最後加上 backdrop-filter 創造立體感。
02

資料呈現：Bento 式排版

資訊拆解：把生硬的人口、房價數據抓出來，單獨做成卡片，加上雙色進度條作為點綴，降低資料閱讀的枯燥感。
實作方式：仰賴 CSS Grid 佈局，利用 grid-column: span 編排卡片大小，配上微透明邊框。
03

購屋情境：預算實戰轉換

情境帶入：把單坪的房價轉化為有實感的購物情境：「預算兩千萬在北屯買小兩房、在竹東買四房」，比單純列出房價更有感。
實作方式：使用雙欄排列，並以偽元素 ::before 安排分隔線與標籤。項目使用原生 <ul> 搭配客製樣式。
04

在地探索：地圖 API 串接

空間連結：放入可縮放的動態地圖代替截圖；點擊目的地卡片，地圖就會自動平滑移動並開啟該標籤的照片與說明。
實作方式：串接 Leaflet.js。自己將座標寫死在陣列裡，用 JS 在地圖上畫線，並控制中心點的 flyTo 飛越特效。
05

網頁收尾：真實數據連動

參與感營造：投票功能不再只是前端動畫，而是真正串接了資料庫，讓讀者點選後能即時看見所有社群參與者的真實意向。
實作方式：後端利用 Google Apps Script (GAS) 將 Google 試算表轉化為免費 API。前端透過非同步 fetch 收發資料並更新進度條。
06

歷史區塊：時間軸設計統一

體驗一致性：為了讓主專案的「歷史發展軌跡」也能擁有高級感，我們將設計紀實中的「現代化垂直時間軸 (Modern Timeline)」封裝並共用至首頁。
實作方式：將時間軸 CSS 樣式抽離至 style.css，並新增 .beitun-timeline 與 .zhudong-timeline 樣式，讓發光節點與卡片邊框能隨地區動態切換專屬品牌色。
