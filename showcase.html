探索本網站的設計與巧思

歡迎來到幕後介紹頁面！「北屯區 vs 竹東鎮 — 雙區魅力對決」不只是一個提供數據的網站，我們更希望透過精心設計的視覺與互動，讓您在瀏覽時感覺像在操作一款精美的 App。以下為您一一介紹我們是怎麼做到的！

視覺與設計美學

如夢似幻的「液態毛玻璃」背景

您是否注意到網站背景不是單調的死白？我們在畫面最底層設計了會緩慢漂浮、交融的色彩光球，並在上方蓋了一層「毛玻璃」效果。這種設計常見於蘋果（Apple）的最新產品介面中，能讓網站看起來充滿生命力，同時又不會搶走文字內容的焦點，帶來頂級的視覺享受。

日式「便當盒」整齊排版

在比較兩個地區時，通常會有密密麻麻的數字與文字。我們採用了近年非常流行的「便當盒（Bento）」排版法，把複雜的資訊切分成一塊一塊大小不一、邊角圓潤的卡片。這樣不但畫面看起來乾淨俐落，在手機螢幕上看也能自動排列得非常完美。

貼心的「日夜模式」一鍵切換

考慮到大家可能在晚上瀏覽網站，我們在導覽列右側設計了太陽/月亮按鈕。點擊後，不只是背景變黑、文字變白，包含圖表顏色、甚至地圖的底圖風格都會瞬間跟著改變！這能保護您的眼睛，也讓網站在夜晚看起來更具科技感。

豐富的互動與實用功能

一目了然的「雷達圖」對比

買房或居住最看重「生活機能」、「房價」、「交通」等指標。我們沒有給您枯燥的表格，而是做了一個動態的六角形雷達圖。您可以一眼看出北屯區的強項在哪、竹東鎮的優勢在哪，兩張網疊加在一起，強弱差距立刻浮現。

真實的地圖與交通探索

我們直接把真實世界的地圖搬進了網站！您不僅可以看到捷運綠線、台鐵內灣線的路線是怎麼跑的，還可以點擊地圖上的地標，畫面會自動滑順地飛過去，並且彈出該地點的照片與詳細介紹，讓您有身歷其境的感覺。

為您量身打造的推薦

每個人對居住的需求都不一樣。因此我們設計了「生活型態選擇器」：無論您是喜歡兩人世界的頂客族、重視空間的小家庭，還是準備養老的退休族，只要點擊對應的卡片，系統就會動態為您推薦最適合的區域，並給出專屬的建議。

即時的社群投票體驗

在網站的最下方，我們設計了一個互動式的投票區域。當您做出選擇後，選項會流暢地轉換為進度條，並搭配數字跳動的動畫，為這趟雙區探索之旅畫下一個充滿參與感的句點。

核心區塊設計理念與架構

針對 index.html 中各大核心區塊，為您深度剖析其背後的設計思維與前端實作技術。

01

首頁：旗艦級滿版雙拼

設計理念：以左右分割的滿版畫面呈現「對決」氛圍，第一眼即確立雙區爭霸的主題。搭配滑順的文字進場動畫，營造頂級發佈會般的氣勢。
架構技術：採用 Flexbox 達成響應式對半佈局。底層運用 liquid-bg 液態光球，上層疊加毛玻璃遮罩 (backdrop-filter)，創造深邃的立體層次。
02

數據對決：Bento UI 便當盒排版

設計理念：捨棄傳統枯燥的數據表格，將人口、房價等統計數字卡片化。透過大小不一的網格與視覺化雙色進度條，讓資訊閱讀直覺且現代化。
架構技術：高度依賴 CSS Grid 的排列與 grid-column: span 屬性。結合 Serif 襯線體呈現數字美學，並加入微透光的玻璃質感。
03

預算購屋：2000萬雙區比較

設計理念：將抽象的房價數據轉化為具體的「購屋實戰」。利用精緻的卡片對比北屯小二房與竹東大四房，讓使用者對預算的購買力有真實感受。
架構技術：利用 housing-grid 設定雙欄佈局，中間使用偽元素或絕對定位放置 VS 標誌。卡片內則結合列表 (ul) 與自訂標籤 (tag) 呈現重點資訊。
04

專屬生活型態：互動式 Persona

設計理念：從「以人為本」出發，設計竹科新貴、頂客族、退休族三種人物誌。讓使用者能代入情境，獲得客製化的生活建議，增強帶入感。
架構技術：運用 JavaScript 監聽卡片點擊事件，動態置換下方結果區塊 (#personaResult) 的 DOM 結構與內容，並加入淡入動畫提升回饋感。
05

在地探索：整合真實互動地圖

設計理念：提供可互動的真實地圖取代靜態截圖。使用者能自由開關捷運、台鐵等交通路線，點擊地標還能查看詳細圖文，創造沉浸式探索體驗。
架構技術：引入 Leaflet.js 開源地圖套件。透過 JS 動態載入經緯度、繪製路線 (Polyline) 與圖標 (Marker)。資訊面板則採用絕對定位覆蓋於右側。
06

動態通勤時間：視覺化賽車跑道

設計理念：將抽象的通勤時間轉化為直覺的「長條圖競賽」。當改變目的地時，長條圖如賽車般動態伸長縮短，讓比較過程不再單調。
架構技術：藉由 HTML <select> 觸發 JS 更新邏輯，計算對應數據後改變 CSS width 屬性，搭配 transition 屬性達成平滑的伸展過場。
07

總結與投票：雷達圖與即時開票

設計理念：以六角雷達圖精準呈現兩區綜合能力。最後的投票區塊將單向資訊轉為雙向互動，讓使用者在評估所有數據後，表達最終的居住意向。
架構技術：雷達圖由 Chart.js 渲染並支援響應式。投票系統則利用 JS 模擬投票送出，動態計算比例並調整進度條寬度，呈現真實的開票效果。

技術與幕後秘密

「沒有使用笨重的框架，打造極致流暢的體驗」

或許您會好奇，這麼多動態效果，網頁會不會很慢？答案是不會！
為了追求最快的載入速度與最滑順的動畫，這個網站沒有依賴任何龐大的網頁生成工具。我們運用了網頁最基礎的語言（HTML、CSS 與 JavaScript），親手刻畫每一個排版與按鈕。

此外，網站上所有的圖示（包含您現在看到的小圖示）都不是一般的圖片，而是使用了先進的「向量線條圖示（SVG）」。這意味著無論您用多麼高解析度的螢幕（如蘋果的 Retina 螢幕）觀看，或是放大畫面，這些圖示永遠都保持清晰銳利，絕對不會有模糊的鋸齒。
